一、 解决拓展性：如何设计“小马跑道”的架构？
如果你未来要在“交互设计”分类里再塞入 3 个新作品，跑道必须能自动延长。我们需要采用**“乐高积木拼装”**的思维：
方案：模块化轨道 (Modular Track Segment)
不要去画一根长达 10000px 的总线。我们应该设计一个**“标准轨道模块”**（比如宽 100vw，高 100vh）。
入口与出口固定： 每一块画布的 SVG 曲线，无论中间怎么绕，它的起点永远在屏幕左侧中间 (0%, 50%)，终点永远在右侧中间 (100%, 50%)。
无限拼接： 当你把 10 个作品组件横向拼在一起时，这 10 根短线在视觉上就会严丝合缝地连成一条无限长、平滑波动的长线。
数据映射： 在你的代码里，只需要维护一个 JSON 数组 const works = [{ title: 'A', category: 'UI' }, { title: 'B'... }]。React 每渲染一个对象，就自动在后面增加一节轨道。未来无论你怎么加作品，轨道永远不会断，滚动距离由 GSAP 自动重新计算。
分类跳转怎么做？(分支轨道)
如果小马正在主线（所有作品）上跑，你想快速看“视觉设计”分类怎么办？
在屏幕顶部固定一个“分类导航”。当点击“视觉设计”时，利用 GSAP 的 scrollTo，让屏幕瞬间极速横向平移，小马就像开启了**“曲速引擎”**一样，瞬间飙车滑动到视觉设计的第一节轨道。
二、 开发流程：千万不要一步到位！
做这种复杂的连续 WebGL/SVG 交互，如果直接让 AI 给你写最终形态的代码，99.9% 会直接崩盘。你需要极其严格地按照以下 4 个阶段（从 MVP 到 Final）来推进一步一步的代码。
🚧 Phase 1: MVP 数据与横向卷轴 (The Cargo Train)
目标： 不加任何花里胡哨的线和小马，只实现基于 projects.json 数据生成的无限横向滚动劫持（Horizontal Scroll-jacking）。
验证： 数组里加减数据，看看横向滚动的尽头会不会出错，确保底盘极度稳定。
🛤️ Phase 2: 铺设模块化轨道 (The Seamless SVG)
目标： 在每个作品的背景里，加入我们刚才说的“起点终点固定”的 SVG 曲线。
交互： 使用之前教你的 stroke-dashoffset，让这根线随着横向滚动的进度，逐渐从左向右被“画”出来。
🐎 Phase 3: 绑定小马与动态路径 (MotionPath Integration)
目标： 引入 GSAP 的 MotionPathPlugin。
逻辑： 既然我们有了一根连贯的线，我们将一个小马（可以是一个发光的 SVG，或者一个 3D 模型渲染出的 GIF/WebM）吸附在屏幕绝对居中的 X 轴上，然后通过读取当前滚动的进度，计算出它在 Y 轴（高度）上的起伏。让它看起来像是在顺着线飞。
✨ Phase 4: 终极打磨 (The Unseen Polish)
目标： 加入作品的 Hover 悬停放大、背景视频的自动播放、以及小马在经过特别节点时的粒子特效爆发。

2. 构建横向卷轴 DOM 结构
最外层创建一个触发器 <section id="works-trigger">，高度设为 h-[300vh]（动态计算更好，暂时给个足够高的高度提供滚动空间）。

3. 编写横向滚动劫持 (GSAP ScrollTrigger)
⚠️ 关键要求：必须根据动态数据的长度来计算平移距离！
code
JavaScript
useGSAP(() => {
  const track = document.querySelector('#works-track');
  
  // 动态计算：总平移距离 = 轨道总宽度 - 单个屏幕宽度
  const scrollWidth = track.scrollWidth - window.innerWidth;

  gsap.to(track, {
    x: -scrollWidth, // 向左平移
    ease: 'none', // 必须是 none，保证横向匀速
    scrollTrigger: {
      trigger: '#works-trigger',
      start: 'top top',
      end: () => `+=${scrollWidth}`, // 滚动距离与内容真实长度 1:1 映射
      pin: true,
      scrub: 1, // 平滑跟随
      invalidateOnRefresh: true, // 响应式窗口大小改变时重新计算
    }
  });
},
