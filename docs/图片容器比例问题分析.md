# 图片容器比例问题分析

## 现象

案例二（无忧萌环）图二 `image_157.png` 反复出现上下被裁剪、显示不完全的问题。

## 根因

### 1. vw 和 vh 混用导致容器比例不可控

原配置：

```
width: "84vw", height: "42vh"
```

在 1920×1080 屏幕上：
- 容器实际像素：`84 × 19.2` = 1613px 宽，`42 × 10.8` = 454px 高
- 容器宽高比：1613 / 454 ≈ **3.55:1**

图片 `image_157.png` 原始尺寸：2120 × 1204，宽高比 **1.76:1**

容器 3.55:1 vs 图片 1.76:1——相差接近一倍。`object-cover` 为了填满容器，将图片等比放大到宽度填满（1613px），然后高度只能显示中间 454px，上下大量内容被裁掉。

### 2. 屏幕尺寸影响

vw/vh 的像素值随屏幕变化，容器宽高比是**不稳定**的：

| 屏幕 | 1vw 像素 | 1vh 像素 | 84vw×42vh 实际宽高比 |
|------|---------|---------|---------------------|
| 1920×1080 | 19.2px | 10.8px | 3.55:1 |
| 1440×900 | 14.4px | 9.0px | 3.73:1 |
| 2560×1440 | 25.6px | 14.4px | 3.55:1 |

不同屏幕上的容器比例差异巨大，但图片比例始终是 1.76:1。

## 解决方案

**宽高统一使用 vw**，容器比例在任何屏幕上与图片比例一致：

```
width: "28vw", height: "15.9vw"
```

换算：`28 / 15.9 ≈ 1.76`，与图片 2120/1204 ≈ 1.76 匹配。

| 屏幕 | 28vw 像素 | 15.9vw 像素 | 容器宽高比 |
|------|----------|-----------|-----------|
| 1920×1080 | 537.6px | 305.3px | **1.76:1** |
| 1440×900 | 403.2px | 229.0px | **1.76:1** |
| 2560×1440 | 716.8px | 407.0px | **1.76:1** |

## 规则总结

| 规则 | 说明 |
|------|------|
| **宽高统一单位** | 图片容器宽高必须用同一 CSS 单位（都是 vw，或都是 vh，或都是 px） |
| **vw+vh 混用 = 比例失控** | `vw / vh` 的值随屏幕宽高比变化，同一组数字在不同屏幕上是完全不同比例 |
| **用 vw 适合横向滚动** | 因为 Stage 4 是横向滚动布局，用 vw 确保图片随滚动等比缩放 |
| **比例公式** | `height_vw = width_vw × (image_height_px / image_width_px)` |

## 适用范围

此规则适用于**所有** `extraImages` 和 `imageConfig` 中的尺寸配置。
