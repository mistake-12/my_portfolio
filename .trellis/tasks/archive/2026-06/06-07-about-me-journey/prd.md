# About Me：跨界者的寻宝轨道

## Goal

重新设计 Stage 4 轨道上 `category-about` 区域。5 张散落卡片沿轨道错落排列，提炼跨界历程中的经验与能力，配合小鸟旁白气泡营造"寻宝旅程"叙事感。

## Requirements

* 5 张卡片沿轨道上下错落排列，替换 blank CategoryIntro 占位
* 卡片轻倾斜（±2-5°）、玻璃质感、浮动动画
* 卡片内容提炼能力/洞察，不列项目名
* 小鸟飞到卡片附近时自动弹出第一人称旁白
* 第 5 张卡片含 GitHub 图标，点击跳转 github.com/mistake-12
* Stage 2 AboutContent 保留不动
* 保持云朵装饰

## 卡片文案

### 🏗️ 建筑（上方, tilt: -2.5°）
**描述**：两年建筑训练给了我结构思维——先理解力的传递路径，再赋予形态。从物理模型的破坏实验中学会了：优雅的形态源于诚实的受力。空间感、尺度感、构造逻辑，这些成了我理解一切复杂系统的底层框架。
**标签**：`结构思维` `系统视角` `形式追随力`
**小鸟旁白**："那两年教会我：好的设计自己会站立，不需要多余的装饰。"

### 🎨 工业设计（下方, tilt: 3°）
**描述**：从建筑转到工业设计，尺度从城市缩小到手掌，但追问的方向反了过来——不再从形式出发，而是从"谁在用、在什么场景、有什么感受"出发。学会了用同理心定义问题，用原型验证假设，在用户、技术与商业之间找到平衡点。
**标签**：`用户共情` `问题定义` `原型验证`
**小鸟旁白**："从造房子到造产品，尺度变了，但那个追问始终没变——它真的让人的生活变好了一点吗？"

### 🧠 个人能力（上方, tilt: -3°）
**描述**：能拆解复杂业务——把模糊的"不好用"翻译成可执行的产品方案，把 4 层信息架构压到 2 层。能从 0 到 1 定义 AI 产品——识别真实场景、界定 MVP 边界、用原型快速验证、推动上线并追踪数据闭环。能全栈独立交付——不依赖外部资源，一个人完成从数据库设计到前端动效的完整产品。
**标签**：`业务拆解` `产品定义` `全栈交付`
**小鸟旁白**："我的核心能力就一条：拿到一个模糊的问题，给出一套能跑的方案。"

### 🛠️ 技术栈（下方, tilt: -2°）
**描述**：设计：Figma · Blender · Rhino · KeyShot。前端：React · Next.js · TypeScript · Tailwind CSS · GSAP · Vue 3。后端：Node.js · PHP(ThinkPHP) · Supabase · WebSocket。AI：Agent 工作流设计 · RAG · LLM 应用。

**小鸟旁白**："设计让我发现问题，代码让我解决问题——两者加起来才完整。"

### 📮 联系我（上方, tilt: 2.5°）
**描述**：对上海、深圳、杭州的机会持开放态度。欢迎通过 GitHub 了解我的更多项目，或直接联系交流。
**标签**：`Open to Work`
**链接**：GitHub 图标 → https://github.com/mistake-12
**小鸟旁白**："如果我的经历让你感兴趣——来找我聊聊吧！"

## 视觉设计

* 宽度：26vw / 高度自适应
* 倾斜：每张卡 ±2-5°，交替方向
* 质感：`rgba(255,255,255,0.08)` + `backdrop-filter: blur(12px)` + `border: 1px solid rgba(255,255,255,0.1)`
* 底部 1px `#FF4D00` 细线（呼应实线轨道）
* 浮动动画：`card-float` 4-6s ease-in-out，每张不同延迟
* 卡片与轨道间距：60-80px
